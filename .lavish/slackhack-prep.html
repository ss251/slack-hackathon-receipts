Build half — the discipline
  • ◆48h spike gate ("the cake"): prove the single scariest, most-magical call end-to-end before committing. GO/NO-GO.
  • ⟲AGENT_STATE.md loop spine: read-file → do first unchecked item → log → repeat. Survives context compaction.
  • ▲Scope: primitive survives, shell is disposable. Pre-plan degrade paths; cut ruthlessly.
  • ✕Seed a deliberately-broken state so the failure/edge story is a permanent receipt, not a claim.
  • ▶Prove it live, then write the README from receipts. Submission materials come LAST, after the product is at its best.
02

The Slack target, decoded

Build a Slack agent that solves a real, specific workflow inside Slack. The rules explicitly call out the losing pattern: "a generic chatbot wrapped in a Slack UI." Avoid that lane.

Use ≥1 required tech "in a way that wouldn't be possible without it"
◆ Recommended · hacker path

MCP + RTS API

  • RTS = permission-aware live search of the real workspace's threads/files
  • MCP = Slack actions as tools an LLM invokes; OAuth auto-handled
  • Together: search → reason → act in Slack's native permission model
  • Bolt app; user token, scope search:read.public+
low-code path

Agent Builder / Agentforce

  • Custom Slack actions via Apex @InvocableMethod
  • Needs a Salesforce org + Named Credentials
  • Fastest for CRM-adjacent / enterprise flows
  • Heavier setup; less "look what only Slack can do"
the two tracks

Good vs Orgs

  • Agent for Good — judged on measurable social benefit
  • Agent for Organizations — judged on real adoption potential
  • Chosen at submission — bias toward the thinner credible field
Judging bar — 3 criteria
  1. Solves a real, specific Slack workflow (not a chatbot skin)
  2. Required tech used in a way not otherwise possible
  3. Clear impact — social benefit (Good) / adoption (Orgs)
Judges spend ~5–7 min/project; demo video hard-capped at < 3 min — design criteria 1 & 2 to land on the same screen.
Submission checklist
  • Demo video < 3 min (YouTube/Vimeo), showing it actually working
  • Architecture diagram
  • Slack developer sandbox URL
  • Test access to slackhack@salesforce.com + testing@devpost.com
  • No unlicensed trademarks / copyrighted material
De-risked (SDK-Skeptic check passed): the free Slack Developer Program sandbox is a full Enterprise Grid org ($0; 2 sandboxes × 3 workspaces × 8 users; 6-mo life) — it unlocks the otherwise paid-gated RTS API, MCP, and Agent Builder. A solo dev can build the whole agentic path for free. This is Rally's "free testnet," and it greenlights the MCP+RTS hacker path.
