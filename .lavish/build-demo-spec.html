Contradiction Catcher · build & demo spec

What we're actually building — and how it demos

Stripped to the metal: a small event-driven Slack app that, when you "lock" a decision, does one permission-scoped live search and posts a warning if it collides with a prior decision you're allowed to see. No new database, no index, ~1 search + 1 LLM call + 2 writes.

✅ The load-bearing claim is confirmed by Slack's own docs. RTS (assistant.search.context) "performs the search on behalf of the authenticated user, ensuring that only content the user has access to is returned." The ACL boundary — the whole demo — is a property of the API, not something we fake.
01

In one breath

A teammate reacts :lock: on a message that settles a decision. The app wakes up, searches the workspace as that person for a conflicting earlier decision, and if it finds one, replies in-thread: "⚠️ this contradicts @Dana's locked call in #product-private [jump →]." A conflicting decision sitting in a channel that person can't see is silently, correctly ignored.

Why it reads as an "agent," not a chatbot: nobody types at it. It perceives (the lock), retrieves (RTS), reasons (is this a reversal or a refinement?), and acts (warn + reconcile) — autonomously, inside a real workflow, respecting permissions. That's the exact "generic chatbot in a Slack UI" trap, avoided.

7 · The ACL boundary (zero code)
A conflicting decision in a channel the locker isn't in is never returned by step 4 — so it's never surfaced. We write nothing to make this happen; it falls out of RTS searching as the user. This is the unfakeable, only-possible-with-RTS moment.
emergent from RTS permission model
The stack (deliberately small)
Bolt for TypeScript Socket Mode (demo) · Railway (judges) RTS API — the required tech Claude — keyword + classifier Web API — post + canvas
Honest simplification (de-risks the build)
  • 1RTS alone satisfies the required-tech gate. MCP and Agent Builder are optional — build the canvas with plain canvases.create for reliability, add MCP only if time allows (stacks a 2nd pillar).
  • 2Keyword mode only. A natural-language question flips RTS into non-deterministic semantic mode — never phrase the query as a question.
  • 3No new datastore. RTS forbids storing its data — so we classify in-memory, surface via permalink, and persist nothing. That's simpler, not harder.

04

The honest risks (and what the spike proves)

Nothing here is a showstopper, but two are real unknowns the 48h spike must clear before we build polish. I'd rather name them now.

Risk 1 — identity mechanics. To search "as the locker" across private channels we need either their user token (xoxp + search:read.private) or a bot token + the action_token from the event. Which triggers carry an action_token is the #1 thing to verify.
Spike hour 1. If the :lock: reaction doesn't carry an action_token, switch the trigger to a message shortcut / app-mention that does, or do a one-time user-token OAuth (fine for the demo — Alex authorizes once).
Risk 2 — the "no-store" rule. RTS says you must not store/copy its data. Copying a prior decision's full text into a persisted canvas is a gray area.
Design around it. Surface via permalink + a short quote (Slack calls sharing sources a best practice), don't persist retrieved content in our own store. Simpler build, cleanly compliant.
Risk 3 — classifier determinism. An LLM deciding "is this a reversal" can misfire (~4% residual).
Temp 0 + stark seed + few-shot + record a fallback take. On scripted Mongo↔Postgres contrast it's effectively deterministic.
Risk 4 — filming the negative control. To show the hidden Cassandra conflict was correctly suppressed, the judge must believe it exists.
Plan the second vantage: a workspace-owner/admin account (sandbox gives you 8 users) that can see #platform-infra — quick cut to it in the recording.
Bottom line: the core loop (lock → search-as-you → warn + jump) is small and firmly buildable — Slack's docs confirm the ACL property that makes it special. The two genuine unknowns (identity mechanics, RTS result quality in keyword mode) are exactly what the 48h cake spike exists to settle. If RTS underwhelms, the graceful pivot is Leaving Brain (search→render, no classifier gamble).
