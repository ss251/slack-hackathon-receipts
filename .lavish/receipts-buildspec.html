Architecture — small on purpose

Bolt app, one RTS search, one Claude call, one post. No datastore (the FAQ canvas is the only persistent artifact). Every arrow is a real API.

Summon
@Receipts <q> (app_mention — the reliable action_token carrier) or a "Pull receipts" message shortcut.
Bolt · app_mention

Read intent + keyword
Claude (temp 0) extracts the question and the rare distinctive topic token.

Search history AS the summoner
assistant.search.context, keyword mode, incl. private channels — returns prior messages with author, ts, permalink. Permission-scoped to the summoner.
RTS · the engine
4
Rank + repeat-detect
Claude picks the best prior decision/answer; N hits = "asked N×"; "what's changed: nothing" if nothing later contradicts it.
Claude · temp 0

Block Kit card: verbatim quote + @author + date + Jump button. Whisper-first option to save the asker's face.
chat.postMessage
6
Deflect → FAQ canvas
On a repeat, offer "add to FAQ"; maintainer approves → append to the canvas that grows as a side effect.

The ACL boundary (zero code)
A decision in a channel the summoner isn't in never returns from step 3 — so it's never leaked. Falls out of searching as the user. The unfakeable "only-possible-with-RTS" artifact.

04

Do FIRST — the 48h cake spike (go/no-go)

✅ Must be TRUE on seeded sandbox data before any polish
  • 1Identity — search AS the summoner. Prove app_mention delivers a usable action_token for RTS bot-token search. #1 RISK If not → one-time user-token OAuth, or degrade to public-channel bot-scope search.
  • 2Retrieval. A keyword search reliably returns the seeded prior decision from conversation history with a resolvable permalink.
  • 3ACL boundary. A decision in a channel the summoner is NOT in is correctly not surfaced — the unfakeable artifact. GO/NO-GO here.
Fallback if it wobbles: if search-as-summoner is hard, ship public-channel bot-scope search — loses the ACL kicker but still works for community channels (most of the value). If retrieval is noisy on messy archives, show top-3 candidate links, never a synthesized answer without receipts.
